面向对象中的不同

首先说明，这篇比较是针对拥有基本的C++面向对象基础，而不得不学习Java的同学的。

类的声明

首先在C++中声明类的时候为:

class A{

//我是仅存且不存在的内容

};

然鹅，当用Java声明类的时候，你会发现，Java需要声明类的类型。(即public)

public class A {

//我是仅存且不存在的内容

}

成员的声明

  • 首先要说明是不论是Java还是C++中，如果不标明成员的类型，默认都为私有的。
  • 在C++中可以把成员类型的定义可以看成是一次性的事,而在Java中是更清晰的事情。
    在C++中类成员定义:
class A{

private:

    int numb1;

    char numb2;

public:

    int getNumb1(){

        return numb1;

    }

};

首先要说明的public中的部分成分还没有完善，这里个人认为不是声明的重点，所以略过。

Java中成员的声明:

public class student {

//我是仅存且不存在的内容

    private int numb1;

    private char numb2;

    public int getNumb1() {

        return numb1;

    }

}

类的封装

首先说明的是我学识浅薄，只知道在Java中封装是必须，或者说是一种习惯，并不知道在C++中，封装是否为约定俗称的习惯。

封装就是为private的数据类型设置相应的get\set函数，通过函数的方法，来保护数据的安全，以符合面向对象的编程思想。

在Java中，封装:

public class student {

//我是仅存且不存在的内容

    private int numb1;

    private char numb2;

    public int getNumb1() {

        return numb1;

    }



    public void setNumb1(int numb1) {

        this.numb1 = numb1;

    }



    public char getNumb2() {

        return numb2;

    }



    public void setNumb2(char numb2) {

        this.numb2 = numb2;

    }

}

当面对C++时也可以设置相应的get/set函数。

class A{

private:

    int numb1;

public:

    int getNumb1(){

        return numb1;

    }

    void setNumb1(int numb1){

        this->numb1 = numb1;

    }

};

构造函数

C++和Java中都存在构造函数
C++:

class A{

private:

    int part;

public:

    A(){

        //在这可以写构造函数的内容

        

    };

}

Java:

public class student {

//我是仅存且不存在的内容

    private int numb1;

    private char numb2;

    student(){



    }

这里的的student函数也为公有函数

析构函数

首先要说Java没有
得益于Java自己对内存的管理机制，实现了对内存的有效调控(有总比没有强)。

C++:

class A{

private:

    int numb1;

public:

    A(){

        //在这可以写构造函数的内容

    };

    ~A(){

        //再见了，您~      

    };

};
